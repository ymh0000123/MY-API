API 文档

当前服务提供 IP 信息、和风天气、格点天气和 Hypixel 玩家查询。天气类接口会校验 apikey， 成功时直接返回上游数据，失败时返回统一 JSON 错误对象。

基础路径 https://你的域名
响应格式 JSON
天气鉴权 apikey 或 x-api-key

鉴权

天气类接口必须携带 apikey。以下写法都支持：

GET /api/weather?apikey=你的密钥
GET /api/weather?apiKey=你的密钥
GET /api/weather?api_key=你的密钥
x-api-key: 你的密钥
Authorization: Bearer 你的密钥

服务端需要配置 HF_API_KEY，并绑定 D1 数据库 API_KEYS。数据库中有效密钥需要满足 active = 1。

天气位置参数

城市天气接口支持城市名、LocationID 或经纬度；格点天气必须使用经纬度。

参数 适用接口 说明
location=北京 城市天气 城市名、LocationID 或 经度,纬度。
location=116.41,39.92 城市天气、格点天气 经度在前，纬度在后。
lon=116.41&lat=39.92 城市天气、格点天气 等价于传入 location=116.41,39.92。
不传位置 城市天气 优先使用 Cloudflare 经纬度，其次使用 Cloudflare 城市信息。

IP 查询

GET /api/ip
无需鉴权

返回当前请求的 IP 地址和 Cloudflare 识别到的地理位置。

响应示例

{
  "ip": "203.0.113.10",
  "location": {
    "city": "Singapore",
    "region": "Central Singapore",
    "country": "SG"
  }
}

实时天气

GET /api/weather
需要 apikey

查询城市实时天气。旧路径 /api/tq 仍然可用。

请求示例

GET /api/weather?apikey=你的密钥&location=北京

参数

名称 必填 说明
apikey 是 也可通过 x-api-key 或 Bearer Token 传入。
location 否 城市名、LocationID 或经纬度。
lang 否 返回语言，默认 zh。
unit 否 单位制，例如 m 或 i。

每日天气预报

GET /api/weather-daily
需要 apikey

返回城市每日天气预报，默认 3 天。

请求示例

GET /api/weather-daily?apikey=你的密钥&location=北京&days=7d

参数

名称 必填 说明
days 否 可选 3d、7d、10d、15d、30d，默认 3d。
location 否 城市名、LocationID 或经纬度。
lang / unit 否 语言和单位制。

逐小时天气预报

GET /api/weather-hourly
需要 apikey

返回城市逐小时天气预报，默认 24 小时。

请求示例

GET /api/weather-hourly?apikey=你的密钥&location=北京&hours=72h

参数

名称 必填 说明
hours 否 可选 24h、72h、168h，默认 24h。
location 否 城市名、LocationID 或经纬度。
lang / unit 否 语言和单位制。

格点实时天气

GET /api/grid-weather
需要 apikey

查询指定经纬度的格点实时天气。

请求示例

GET /api/grid-weather?apikey=你的密钥&location=116.41,39.92

格点天气必须提供经纬度，推荐使用 location=经度,纬度。

格点每日预报

GET /api/grid-weather-daily
需要 apikey

返回指定经纬度的格点每日天气预报，默认 3 天。

请求示例

GET /api/grid-weather-daily?apikey=你的密钥&location=116.41,39.92&days=7d

参数

名称 必填 说明
location 或 lon + lat 是 经纬度，格式为 经度,纬度。
days 否 可选 3d、7d，默认 3d。
lang / unit 否 语言和单位制。

格点逐小时预报

GET /api/grid-weather-hourly
需要 apikey

返回指定经纬度的格点逐小时天气预报，默认 24 小时。

请求示例

GET /api/grid-weather-hourly?apikey=你的密钥&location=116.41,39.92&hours=24h

参数

名称 必填 说明
location 或 lon + lat 是 经纬度，格式为 经度,纬度。
hours 否 可选 24h、72h，默认 24h。
lang / unit 否 语言和单位制。

Hypixel 玩家查询

GET /api/hyp
无需 apikey

根据 Minecraft 玩家 UUID 查询 Hypixel 玩家信息。

请求示例

GET /api/hyp?uuid=玩家UUID

参数

名称 必填 说明
uuid 是 Minecraft 玩家 UUID。

错误格式

接口失败时会返回统一结构，调用方可以读取 error.code 和 error.message。

{
  "success": false,
  "error": {
    "code": "MISSING_API_KEY",
    "message": "缺少 apikey。请使用 ?apikey=你的密钥，或通过 x-api-key 请求头传入。"
  }
}
