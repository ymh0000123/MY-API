文档

这里列出当前可用 API 的请求方式、参数和返回示例。所有接口在出错时都会返回 JSON，方便调用方直接读取错误原因。

IP查询

GET /api/ip

无需参数，返回当前请求的 IP 地址和 Cloudflare 识别到的地理位置信息。

返回数据格式


            {
                "ip": "你的IP",
                "location": {
                    "city": "城市",
                    "region": "地区",
                    "country": "国家"
                }
            }

天气查询

GET /api/weather?apikey=你的密钥

兼容旧路径：GET /api/tq?apikey=你的密钥。

根据当前请求的 Cloudflare 城市信息查询实时天气。

apikey 可以放在 URL 查询参数中，也可以放在 x-api-key 请求头中。URL 参数兼容 apikey、apiKey 和 api_key 三种写法。

location 可选。城市天气接口支持 location=城市名、location=LocationID 或 location=经度,纬度；不传时会优先使用 Cloudflare 识别到的经纬度或城市。

服务端需要配置环境变量 HF_API_KEY，并绑定 D1 数据库 API_KEYS 用于校验 apikey。

由于和风天气计费方式调整，该接口必须携带 apikey。

            {
                "code": "200",
                "updateTime": "最近更新时间",
                "fxLink": "响应式页面URL",
                "now": {
                    "obsTime": "数据观测时间",
                    "temp": "温度",
                    "feelsLike": "体感温度",
                    "icon": "和风天气图标码",
                    "text": "天气状况的文字描述",
                    "wind360": "风向360角度",
                    "windDir": "风向",
                    "windScale": "风力等级",
                    "windSpeed": "风速",
                    "humidity": "相对湿度",
                    "precip": "当前小时累计降水量",
                    "pressure": "大气压强",
                    "vis": "能见度",
                    "cloud": "云量",
                    "dew": "露点温度"
                },
                "refer": {
                    "sources": [
                        "天气归属"
                    ],
                    "license": [
                        "和风天气使用协议"
                    ]
                }
            }

错误返回示例


            {
                "success": false,
                "error": {
                    "code": "MISSING_API_KEY",
                    "message": "缺少 apikey。请使用 ?apikey=你的密钥，或通过 x-api-key 请求头传入。"
                }
            }

每日天气预报

GET /api/weather-daily?apikey=你的密钥&location=北京&days=3d

返回城市每日天气预报。days 可选值：3d、7d、10d、15d、30d，默认 3d。

可选参数：location、days、lang、unit。

逐小时天气预报

GET /api/weather-hourly?apikey=你的密钥&location=北京&hours=24h

返回城市逐小时天气预报。hours 可选值：24h、72h、168h，默认 24h。

可选参数：location、hours、lang、unit。

格点实时天气

GET /api/grid-weather?apikey=你的密钥&location=116.41,39.92

返回指定经纬度的格点实时天气。格点天气必须提供经纬度，或让 Cloudflare 提供请求来源经纬度。

可选参数：location、lon、lat、lang、unit。

格点每日天气预报

GET /api/grid-weather-daily?apikey=你的密钥&location=116.41,39.92&days=3d

返回指定经纬度的格点每日天气预报。days 可选值：3d、7d，默认 3d。

可选参数：location、lon、lat、days、lang、unit。

格点逐小时天气预报

GET /api/grid-weather-hourly?apikey=你的密钥&location=116.41,39.92&hours=24h

返回指定经纬度的格点逐小时天气预报。hours 可选值：24h、72h，默认 24h。

可选参数：location、lon、lat、hours、lang、unit。

Hypixel玩家查询

GET /api/hyp?uuid=玩家UUID

根据 Minecraft 玩家 UUID 查询 Hypixel 玩家信息。

服务端需要配置环境变量 HYPIXEL_API_KEY。

返回数据格式


            {
                "success": true,
                "player": {
                    "uuid": "玩家UUID"
                }
            }

错误返回示例


            {
                "success": false,
                "error": {
                    "code": "MISSING_UUID",
                    "message": "缺少 uuid 参数，请使用 /api/hyp?uuid=玩家UUID 进行查询。"
                }
            }
